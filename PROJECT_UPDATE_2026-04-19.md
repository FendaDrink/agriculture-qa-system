# 项目更新说明（2026-04-19）

本次更新覆盖前台小程序、管理后台、业务后端与算法服务，重点完成了「常见问题真实化」「语音交互优化」「城市化管理与权限」「双库检索策略」等能力。

## 1. 前台小程序（frontend）

### 1.1 聊天交互优化
- 新增流式回答“取消发送”能力。
- 输入模式支持“键盘 / 语音”切换（微信式入口）。
- 语音模式支持“按住说话、上滑取消、松开转文字”。
- 语音录制稳定性增强，降低 `operateRecorder: fail NotFoundError` 触发概率。

### 1.2 常见问题页改造
- 由静态假数据改为后端动态接口返回。
- 页面仅展示“问题”，不再展示“答案”。
- 支持手动维护问题 + 自动高频问题的合并展示。

### 1.3 底部 Tab 图标
- 为首页/聊天/常见问答/我的补齐普通态与选中态图标。

## 2. 管理后台（frontend-admin）

### 2.1 常见问题管理
- 新增“常见问题”菜单与页面。
- 支持手动问题 CRUD。
- 自动高频问题支持“编辑、启用/禁用、排序、删除（覆盖禁用）”。
- 支持自动高频问题被手动覆盖后前台去重展示。

### 2.2 用户管理城市化
- 用户新增“所属城市”字段。
- 新增/编辑/列表/筛选均支持按城市维度管理。

### 2.3 向量库管理城市化
- 向量库新增“所属城市”展示与筛选。
- 超级管理员可指定城市创建/维护向量库。
- 管理员创建时城市自动锁定为本人所属城市。

## 3. 业务后端（backend）

### 3.1 FAQ 模块（新增）
- 新增 FAQ 实体、服务、控制器与模块。
- 提供前台列表接口与后台管理接口。
- 自动高频问题来源于真实用户提问聚合。
- 支持自动高频问题“覆盖维护”（originQuestion 机制）。

### 3.2 城市字段与权限
- 用户实体新增 `city` 字段，登录 token 同步携带城市信息。
- 向量库实体新增 `city` 字段。
- 数据库模块接口按角色执行权限约束：
  - 超管：全量可管
  - 管理员：仅本市可管
  - 普通用户：仅本人数据范围

### 3.3 问答召回策略
- 接入“城市路由 + 公共库”双库召回。
- 默认总上下文 `Top5`（合并去重后截断）。
- 召回上下文由后端整理后传给算法服务，减少无效 token 消耗。

## 4. 算法服务（algorithm-rag）

- `main_v2.py` 的 `/completion` 支持接收 `context_chunks`。
- 当收到后端传入上下文时，优先使用传入上下文构建提示词；否则走原有单库检索兜底逻辑。

## 5. 兼容与验证

- backend 构建通过。
- frontend-admin 构建通过。
- frontend TypeScript 检查通过。
